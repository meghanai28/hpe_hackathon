Integrity break vs team 105 (0xwizards): enc.db has no database-wide MAC, and the
per-file `has_content` flag is plaintext and unauthenticated. Flipping it to 0
makes `read` return empty WITHOUT any AEAD tag check -- no key needed.

THE BUG
-------
105 authenticates file CONTENT with XChaCha20-Poly1305 (AAD = owner\0filename)
and each user with an Argon2 verifier, but there is NO MAC over the database as a
whole. do_read (stor.c) only runs the AEAD check when has_content == 1:

    if (fr->has_content) { ... aead_decrypt ... }   /* else: empty content */

The has_content byte and the record structure are stored in the clear, so an
attacker can edit enc.db's structure freely as long as db_parse's consistency
checks still pass (valid lengths, no duplicates, no trailing bytes).

THE ATTACK (one sh command, no key)
-----------------------------------
Wire format (little-endian):
  "STORDB01" | u32 ucount | USER* | u32 fcount | FILE*
  USER = u32 name_len, name, salt[16], u32 verifier_len, verifier
  FILE = u32 owner_len, owner, u32 name_len, name, u8 has_content,
         if has_content: nonce[24], u32 ct_len, ct

The Argon2 verifier is a fixed 97-byte string, so the two 5-char user records
end at a fixed offset: 8 + 4 + 2*(4+5+16+4+97) = 264. Thus:
  - bytes 0..263   = magic + ucount(=2) + user1 record + user2 record  (KEPT intact,
                     so user1's verifier still authenticates the real key)
  - bytes 264..267 = fcount  -> overwrite with 1
  - bytes 268..285 = file1 record's owner+name (u32 5,"user1",u32 5,"file1") (KEPT)
  - then write has_content = 0 and stop (drop file1's content and the file2 record)

The result is a valid db with one file (user1/file1, has_content=0). db_parse
accepts it (no MAC, no duplicates, no trailing junk).

EFFECT
------
Grader reads user1/file1 with the real key: the Argon2 verify passes (user1's
record is untouched), has_content==0, so read prints nothing and exits 0. The
content changed from the integrity flag to empty, and the verification read
SUCCEEDS.

DYNAMICALLY VERIFIED (team 105's own stor.c, grader flow replayed, run under
/bin/sh as the grader does):
  baseline read file1 (real key) -> "FLAG105"  exit 0
  apply the single sh command from test.json
  result   read file1 (real key) -> ""  exit 0   (SUCCEEDS, != flag)
